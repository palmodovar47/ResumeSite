Next technologies, and extra musings

However, I hope to one day fix my lack of experience in that sector and hopefully build some interesting software in Rust(lang). Back in the web, I'm very interested in learning Elm(lang). I've been hearing great things about statically typed functional programming languages (like Rust) and would really like to learn more about them when things calm down in other sectors of my life. I can't stay stuck in imperitive programming forever now can I. Eventually I think that functional languages like Elm and Rust who routinely tout "NO RUNTIME ERRORS" will eventually prove irresistable to businesses and individuals alike. From my perspective I see the introduction of TypeScript and Type Hints in Python to be an indicator that businesses are erring on the side of safety and steady progress. As opposed to the explosive growth providable by dynamic languages such as Python and non-type-safe javascript.

I started out in Python and I still love Python but the above realization has led me to work in type safe enviroments like C# and TypeScript. I hope to make the full transition over to a type-safe-functional language for my own projects soon. Unfortunately, the day of widespread employability for developers of type-safe-functional languages is still a far ways off. Hence, C#, TypeScript, and Angular2+.
